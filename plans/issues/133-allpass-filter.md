# Allpass Filter (#133)

**Doability:** ⭐⭐⭐⭐☆ (4/5) — the core DSP primitive already exists and is unused; only the device shell (schema/adapter/processor/editor) and chained-iteration logic are new.
**Type:** feature
**Scope:** medium

## What is asked
A new effect device: a frequency knob (with an invert button) marking where the allpass response starts, an "amount" knob controlling the filter's Q/resonance, and an "intensity" knob (1-16) controlling how many allpass stages are chained in series, for a phased/comb-like sound.

## Current behaviour / relevant code
- `BiquadCoeff.setAllpassParams(frequency: unitValue, q: number)` already exists at `packages/lib/dsp/src/biquad-coeff.ts:151-172` (2nd-order allpass, same `unitValue = frequency/sampleRate` convention as the other biquad modes). It sits unused — no current device calls it (only `setHighpassParams`/`setLowpassParams` are consumed today, e.g. by Delay and Revamp).
- No `Allpass` device, box, processor, or editor exists anywhere in the repo (`grep -ri "allpass"` only hits `biquad-coeff.ts` and its `dist/` output).
- Processing wrapper: `packages/lib/dsp/src/biquad-processor.ts` (`BiquadMono`/`BiquadStack`). `BiquadStack` (used by Revamp for its 4th-order high/low-pass, `RevampDeviceProcessor.ts:67-68`) is fixed at a hardcoded stage count — for a 1-16 configurable chain, use an array of `BiquadMono` instances sized to the `intensity` field (or generalize `BiquadStack` to accept a variable count) rather than reusing the class as-is.
- Stepped 1-16 field precedent: `CrusherDeviceBox`'s `bits` field is an `Int32Field` with a `{min:1, max:16}` constraint object (not a separate schema "type") — see `packages/studio/boxes/src/CrusherDeviceBox.ts:41-43,223-242` (generated) and the forge schema equivalent in `packages/studio/forge-boxes/src/schema/devices/audio-effects/CrusherDeviceBox.ts`.
- Boolean invert-button precedent: `StereoToolDeviceBox`'s `invertL`/`invertR` fields (`packages/studio/forge-boxes/src/schema/devices/audio-effects/StereoToolDeviceBox.ts:20-21`).
- Frequency knob precedent (exponential 20Hz-20kHz): `packages/studio/forge-boxes/src/schema/devices/audio-effects/RevampDeviceBox.ts:6` (`freqConstraints = {min: 20.0, max: 20_000.0, scaling: "exponential"}`), Q precedent same file line 7 (`qConstraints = {min: 0.01, max: 10.0, scaling: "exponential"}`).

## Plan
1. Schema: new `packages/studio/forge-boxes/src/schema/devices/audio-effects/AllpassDeviceBox.ts` via `DeviceFactory.createAudioEffect`, fields: `frequency` (float32, exponential 20-20000Hz, matching Revamp's `freqConstraints`), `invert` (boolean — inverts the filter's phase response, e.g. negate `b0/b1/b2` or flip the dry/wet sign per the issue's "invert button" on the frequency control), `amount` (float32, exponential Q, matching Revamp's `qConstraints` — this is the "amount of allpass"), `intensity` (int32, `{min:1, max:16}`, default e.g. 1 or 4 — chained stage count, same pattern as Crusher's `bits`).
2. Adapter `AllpassDeviceBoxAdapter.ts`: wrap `frequency`, `amount` as automatable float parameters (`ValueMapping.exponential`, `StringMapping.hertz`/generic), `invert` as a boolean parameter (`ValueMapping.bool`), `intensity` as a raw int32 box field (like `panningMixing` on StereoTool, or `overSampling` on Fold bound via `EditWrapper.forValue` — not necessarily automatable if a stepped control suffices).
3. Processor `AllpassDeviceProcessor.ts`: hold an array of up to 16 `BiquadCoeff`+`BiquadMono` pairs per channel (L/R independent, mirroring Revamp's `processAudio` chaining pattern where each stage's output feeds the next). On `frequency`/`amount`/`invert` change, recompute all active stages' coefficients via `setAllpassParams(frequency/sampleRate, amount)`; apply `invert` as a final polarity flip (multiply output by -1) or by negating the coefficient set — confirm which reading of "invert" the maintainer intends. On `intensity` change, adjust how many of the 16 stages are active in the chain (inactive stages pass through / are skipped, not recomputed each block).
4. Editor `AllpassDeviceEditor.tsx`: knobs for frequency (anchor low, exponential drag like Revamp's band frequency), amount/Q, intensity (stepped 1-16, e.g. a small numeric stepper or short `RadioGroup`/slider), invert as a `Checkbox` button. Optionally a small phase-response or comb-filter curve preview (reuse `Revamp/Renderer.ts`'s canvas approach, plotting phase instead of magnitude, since allpass has unity magnitude everywhere — the interesting visual is phase, not gain).
5. Register in `EffectFactories.ts`, `DeviceProcessorFactory.ts`, `BoxAdapters.ts`, `DeviceEditorFactory.tsx`, `DeviceManualUrls.ts`.
6. WASM mirror: new crate `crates/stock-devices/device-allpass`, porting the same chained-biquad logic (Rust already has the equivalent `BiquadCoeff` port used by `device-revamp`/`device-delay`, just missing an allpass variant — check `crates/dsp/src/biquad*.rs` for the existing coefficient struct and add the allpass coefficient formula there, mirroring `biquad-coeff.ts:151-172` bit-exactly per the project's WASM-parity rules).

## Risks / open questions
- Clarify "invert button" semantics with the maintainer: invert the filter's phase-vs-frequency curve (negate coefficients) vs. simple output polarity flip vs. dry/wet inversion — these produce different sounds and the issue text is ambiguous ("frequency (start of effect) with invert button").
- 16 chained 2nd-order allpass stages per channel is cheap CPU-wise (32 stages total, similar order of magnitude to Revamp's 4th-order stacks), no performance concern expected.
- Confirm whether coefficient recomputation should happen once per stage-count-change (cheap) or if `amount`/`frequency` changes need per-sample smoothing (likely yes, to avoid zipper noise when dragging — mirror Revamp's approach of recomputing on `parameterChanged` at block granularity, which is the existing precedent, not per-sample ramping).
